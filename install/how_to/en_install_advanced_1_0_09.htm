Installation (beta 1.0.09)

Advanced php user installation

NOTE: The following installation procedure is valid ONLY for the beta 1.0.09.

1. Uncompress the CARE 2X beta 1.0.09 scripts into the active directory of your server.

2. Make the following directories and files system writeable.

/cache/barcodes/
/counter/hits/
/counter/hitcount.txt
/fotos/encounter/
/fotos/news/
/fotos/registration/
/logs/ (and all its subdirectories)
/med_depot/img/
/pharma/img/
/radiology/dicom_img/
/gui/img/logos_dept/

3. Enter the host, database name, username, and password information in the include/inc_init_main.php script:

Change the following three secret keys with unique keys of your choice.

$key : The secret key for the first level script lock.
$key_2level : The secret key for the second level script lock.
$ley_login : The secret key for the third level script lock.

4. If you are using a database other than mysql, set the variable $dbtype in the include/inc_db_makelink.php script to the type of your database. The database types are constants as defined in the ADODB manual. Check this link http://php.weblog.com/ADODB to get the db type constant and to check whether your database is currently supported by ADODB 3.11.

5. Enter the domain or ip addresses and hosts information in global_conf/inc_remoteservers_conf.php script.

If you are running Care2x in an SSL server, make sure that the $httprotocol variable in this script is set to "https".

6. Create the database.

7. Run the following sql scripts:

install/sql/care_db_structure_b1009.sql

Note: the following 4 scripts are for the german icd codes! Do not run them if you do not need the german codes.

install/sql/icd10_de/a2g-insert.sql
install/sql/icd10_de/h2n-insert.sql
install/sql/icd10_de/o2s-insert.sql
install/sql/icd10_de/t2z-insert.sql

Note: the following 2 scripts are for the english icd codes! Do not run them if you do not need the english codes.

install/sql/icd10_en/insert-data-a2l.sql
install/sql/icd10_en/insert-data-m2y.sql

Note: the following 3 scripts are for the german ops codes! Do not run them if you do not need the german ops codes.

install/sql/ops301/insert-data-1-5-499.sql
install/sql/ops301/insert-data-55-57.sql
install/sql/ops301/insert-data-58-94.sql

8. Create a new system admin permission.

You need to use the supplied create_admin.php script. Type http://your_host/create_admin.php in your browser. Enter the necessary information and hit the button. Important: After creating the new system admin, remove this script from your active server directory. Leaving it in the active server directory will cause Care2x to block and refuse to run.

9. REMOVE THE FOLLOWING FILES FROM THE CARE2X DIRECTORY!

  • /create_admin.php
  • /install/initialize.php
  • /install/encode_pw_md5.php

    Tip: you can move them to other places outside the web server directory.

    • 10. The installation is finished. Enter the following address in the browser:

    http://your_host/

    (where your_host is the actual host address of your Apache web server). The first page of CARE 2X should appear. Congratulations!

    Now you can start creating user access permissions.

    To do this:

    Select the EDP link on the left menu and then select "Access rights". Enter your admin username and your admin password. When the interface appears, you can start creating user access rights and permissions.

    Note: You cannot create additional Admin permission using this interface. You have to follow the step #8 to create additional admin permissions.
