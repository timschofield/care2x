Installation

Advanced php user installation

NOTE: The following installation procedure is valid ONLY for the beta 1.0.04.

1. Uncompress the CARE 2002 beta 1.0.04 scripts into the active directory of your server.

2. Enter the host, database name, username, and password data in the include/inc_init_main.php script:

Change the following three secret keys with unique keys of your choice.

$key : The secret key for the first level script lock.
$key_2level : The secret key for the second level script lock.
$ley_login : The secret key for the third level script lock.

3. If you are using a database other than mysql, set the variable $dbtype in the include/inc_db_makelink.php script to the type of your database. The database types are constants as defined in the ADODB manual. Check this link http://php.weblog.com/ADODB to get the db type constant and to check whether your database is currently supported by ADODB 3.11.

4. Enter the domain or ip addresses and hosts information in global_conf/inc_remoteservers_conf.php script.

5. Create a database.

6. Run the following sql scripts:

install/sql/care_db_structure_b1004.sql
install/sql/care_db_drg_tables_b1004.sql
install/sql/care_db_create_admin_b1004.sql

Note: the following 4 scripts are for the german icd codes! Do not run them if you do not need the german codes.

install/sql/icd10_de/a2g-insert.sql
install/sql/icd10_de/h2n-insert.sql
install/sql/icd10_de/o2s-insert.sql
install/sql/icd10_de/t2z-insert.sql

Note: the following 2 scripts are for the english icd codes! Do not run them if you do not need the english codes.

install/sql/icd10_en/insert-data-a2l.sql
install/sql/icd10_en/insert-data-m2y.sql

Note: the following 3 scripts are for the german ops codes! Do not run them if you do not need the german ops codes.

install/sql/ops301/insert-data-1-5-499.sql
install/sql/ops301/insert-data-55-57.sql
install/sql/ops301/insert-data-58-94.sql

7. The installation is finished. Point your browser to your installation. The first page of CARE 2002 should appear. Congratulations!

8. CHANGE NOW THE ADMIN ACCESS INFORMATION!

Edit the "care_users" table on the database and change the values of "login_id" and "password" with values of your choice.

Now you can start (if you wish) creating access permission for other users.

To do this:

Select the EDP link on the left menu and then select "Access rights". Enter your admin username and your admin password. When the interface appears, you can start creating user access rights and permissions.

Note: You cannot create additional Admin permission using this interface. You have to use the phpMyAdmin (or any other mySQL administration program) to create additional admin permissions.
