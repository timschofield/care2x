The following are new in the Care 2002 beta 1.0.07 release:

A. Dicom viewer module

  • Search for patients with diagnostic images in Dicom format
  • View the images either in frame or in a separate window
  • View available notes about the image in a separate window
  • The viewer type can be selected by the user (currently only one)
  • Upload diagnostic image files in Dicom format (via HTTP upload)
  • You can upload a single file
  • Or you can simultaneously upload multiple files that belong to a series
  • Number of files to upload is user selectable
  • Write notes to be sent along with the dicom files.
  • Write reference notes to documents, reports, or encounters
  • About the dicom viewer
    • A java applet developed by the Nagoya Institute of Technology
    • Requires a java plug-in to run (downloadable from the Internet)
    • Views 8 - 16 bit gray scale
    • Views 24 bit RGB color
    • Views multiple files as slices in a series
    • Real time WW/WL, Zooming, Magnify, show tag information
    • Image animation, tiled view, invert, flip, rotate
    • B. Enhancement, revisions & corrections

    • The photolab was revised. User navigation is much easier.
    • The patient's label now displays TTF fonts (automatic if TTF font support is available)
    • The patient's label now displays the department, ward ID, room nr.,bed nr., admission type, and insurance type in semi-encoded format
    • Improvements in the registration-admission module
    • Javascript validation of inputs in addition to server side validation
    • Deceased persons are marked with a tiny black cross beside their name
    • Deceased persons cannot be re-admitted anymore (ref: "mutiple-kill" bug)
    • Read-only mode for discharged encounters (to allow review of records)
    • Blood group can now be entered. Blood group is also printed in bold letters on the patient's label.
    • The person's sex is now visualized in lists.
    • Improvements in the ward and outpatient modules
    • The "sex" icons are clickable to pop up a window displaying the persons ID picture
    • Patient search now displays the wards and dates correctly.
    • Direct link from ward's read-only mode to edit-mode (via password check)
  • Link to Care2x ticketed support (OTRS) was added.
  • Link to Care2x newsgroup server mirroring the developers' mailing list was added.

    • C. Pharmacy and Medical Depot Improvements

    • Department can now be selected
    • Order catalog module is now password protected
    • Link to British National Formulary (BNF, a pharmaceutical index) was added
    • Link to "Yellow List" (a german pharmaceutical index) was added

    D. Measurements record improvements

  • "Head circumference" was added to weights and heights
  • "Weights and heights" is now called "Measurements"
  • Data input form was redesigned. All 3 measured values can now be entered simultaneously.
    • E. Operation logbook enhancement
  • Mozilla, Konqueror and other non-MSIE browsers can now move the BP/temp graph forward or back in date, however only one day at a time (MSIE users can additionally right-click on the button to jump to a certain date)

    • F. New language version

    • Romanian language was added (initial files)
    G. Debugs & corrections
    There are lots of bug patches. The most prominent bugs are:

  • Inconsequent language version of department items in combo box.
  • Wrong pending request module when coming from "ambulatory".
  • "Multi-kill" bug. A person could be admitted and discharged as dead ("killed") unlimited times.
  • Past date occupancy bug. Errors in displaying ward occupancy in dates other than present day.
  • Lower/upper case filenames. Most noted in Unix/Linux servers
  • Products' databank no-update bug.
  • Mozilla specific bug in selecting department in pharmacy and medical depot. Mozilla seems to have problems with double header redirects??
  • Blocked records of discharged encounters.
  • eComBill bugs patched by Prakash Yadav.
  • Opera specific bugs. "To-self" links not being compensated by Opera 6.0.
  • Billing types not shown in admission archive search

    Most active bug reporters are Thomas Wiedmann, Sarath, Prakash Yadav & Lukasz Indyk.
