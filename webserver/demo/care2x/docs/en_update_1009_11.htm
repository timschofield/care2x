Updating (beta 1.0.09) to (deployment (mysql) 1.1)

NOTE: The following update procedure is valid ONLY if you have a working 1.0.09 installation.

1. Uncompress the CARE 2002 deployment 1.1 scripts into a temporary directory.

2. Copy all the files into the directory where Care is installed.

Tip: If you have a file manager that compares file dates, use it to copy only the modified files.
3. Open the /include/inc_init_main.php file and enter the previous database access information.

4. Start your phpmyadmin (or any other mysql admin program).

5. Run the following sql script:

install/sql/db_update_1009_11.sql
6. After the conversion of passwords, REMOVE the following files from Care directories:

  • /create_admin.php
  • /install/install.php
  • /install/encode_pw_md5.php
  • /install/initialize.php

    Tip: You can move them to other place outside the web server directory.

    Note: If you want to use the brazilian-portuguese ICD10 codes, create and load the data by doing the following:

    Run the following scripts:

    install/sql/icd10_br/create-table.sql
    install/sql/icd10_br/insert-data-a2l.sql
    install/sql/icd10_br/insert-data-m2z.sql

    Note: I have used the sql script to update the online demo . It worked well. However, it was not an exhaustive test, so please notify me at once if any problem occurs. Thanks. Elpidio
