Conditions needed in updating Deployment 1.1 to 2nd Generation beta 2.0.0
NOTE:

    The 2nd Generation beta 2.0.0 release is not officially recommended to update the existing Deployment 1.1 !

    Inofficially, you can do an update under the following conditions.

A. You have not done any modifications of your own in your current Deployment 1.1 installation

  • Reason: There are lots of scripts that were modified in the 2nd Generation beta 2.0.0, and there is a high level of probability that copying these scripts into your installation will replace your own modifications.

    • B. You are not using the Deployment 1.1 for any real application.

  • Reason: Similar to point A above plus the revisions of the SQL queries for the PostgreSQL support may have introduced a high level of possible bugs.

    • C. You are willing to accept the possibility of experiencing more bugs and possible erratic program behaviour

  • Reason: Same as the above.

    • D. You have to extend the database structure by loading the following sql dump scripts (for mySQL only):

  • http://care2002.sourceforge.net/manuals/update_care_address_citytown.sql
  • http://care2002.sourceforge.net/manuals/create_care_type_immunization.sql

    • I have done own modifications in Deployment 1.1 but I still want to update it with 2nd Generation 2.0.0. Is there a safe way to do it?

      We recommend that you:

    • Copy the 2nd Generation beta 2.0.0 scripts in a separate directory in your web server
    • Edit the /include/inc_init_main.php script and point it to your existing mySQL database for Care2x
    • Load the sql dump scripts listed at point D to extend the database.
    • Run and analyse 2nd Generation beta 2.0.0 and determine which modules and scripts will not conflict with your own modifications.
    • Copy these non-conflicting scripts to your Deployment 1.1 installation.
