Loading ICD & ICPM (OPS) Codes Data

NOTE:You can only load the data from files supplied with this installation package.

A. Do I really need to load these data?

Only if you are using the DRG system or ICD, ICPM (OPS) codes in your documentations.
B. How to start:

Start phpMyAdmin either via the System Admin module or by typing http://yourhostaddress/modules/phpmyadmin/ in your browser.

NOTE: If you like to use a different DBMS administration program, start it instead.

c. What to do next:

There are two methods to load the data from the files:
  • Running SQL insert query scripts
  • Loading files of CSV formatted data

    • D. What are the differences and what method should I use?

    Running sql insert scripts
  • Slow but will work in any standard sql database
  • Use it if your DBMS administration program cannot load a CSV formatted data.

    Loading CSV files

  • Loads the data in high speed
  • You need to know several parameters for it to work
  • If you are using phpMyAdmin or if your DBMS admin program can load CSV data, use this method
    • E. What to do next?

    Follow the method that you have chosen below. (You only need to follow one)
    1. Running sql insert scripts

    If you need the german ICD10 codes:
    Using phpmyAdmin run the following sql scripts in the following order:
    (these are the data for the ICD10 coding system in german language)

  • /install/sql/icd10_de/a2g-insert.sql
  • /install/sql/icd10_de/h2n-insert.sql
  • /install/sql/icd10_de/o2s-insert.sql
  • /install/sql/icd10_de/t2z-insert.sql

    • If you need the english ICD10 codes:
    Similarly run the following sql scripts in the following order:
    (these are the data for the ICD10 coding system in english language)

  • /install/sql/icd10_en/insert-data-a2l.sql
  • /install/sql/icd10_en/insert-data-m2y.sql

    • If you need the german OPS301 (ICPM) codes:
    Similarly run the following sql scripts in the following order:(these are the data for the OPS301 coding system in german language)

  • /install/sql/ops301/insert-data-1-5-499.sql
  • /install/sql/ops301/insert-data-55-57.sql
  • /install/sql/ops301/insert-data-58-94.sql

    • 2. Loading CSV files

    The CSV data have the following format:

    Fields terminated by ; (semi-colon)
    Fields enclosed by " (double-qoute)
    Fields escaped by \ (backslash)
    Lines terminated by \r\n (return,newline chars)
    Field names are NOT included on the first line

    If you need the german ICD10 codes:
    i. Select the care_icd10_de table on the database.
    ii. Load the following text file: (these are the data for the ICD10 coding system in german language)

  • /install/csv/care_icd10_de.csv

    • If you need the english ICD10 codes:
    i. Select the care_icd10_en table on the database.
    ii. Load the following text file:(these are the data for the ICD10 coding system in english language)

  • /install/csv/care_icd10_en.csv

    • If you need the german OPS301 (ICPM) codes:
    i. Select the care_ops301_ed table on the database.
    ii. Load the following text file:(these are the data for the OPS301 coding system in german language)

  • /install/csv/care_ops301_de.csv
